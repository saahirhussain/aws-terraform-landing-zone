# AWS Landing Zone

This project demonstrates the design and implementation of a secure AWS Landing Zone using Terraform and AWS Organizations. The environment establishes a standardized foundation for multi-account AWS deployments by incorporating centralized logging, IAM governance, networking baselines, and security controls.

## Objectives

* Deploy a scalable multi-account AWS environment
* Implement governance through AWS Organizations and Service Control Policies (SCPs)
* Establish centralized logging and monitoring
* Enforce security guardrails and least-privilege access
* Standardize infrastructure deployment through reusable Terraform modules

## Technologies Used

* Terraform
* AWS Organizations
* AWS IAM
* AWS CloudTrail
* AWS Config
* AWS GuardDuty

## Key Features

* Organizational Unit (OU) structure
* SCP enforcement
* Centralized logging account
* Security account architecture
* Reusable Terraform modules
* Cloud governance framework

## Business Value

This project demonstrates enterprise cloud governance, account standardization, security posture management, and Infrastructure as Code best practices commonly used in large-scale AWS environments.
